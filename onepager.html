全体像 — AI営業パイプライン8ステップ

「問い合わせメールが1通届いたところから、商談を経て受注するところまで」を、8つのステップに分解しました。

#ステップ何をするか主に使う技術
①商品設計AI研修・AIコンサルのような商品を、AIとの壁打ちで設計するChatGPT / Claudeでの壁打ち
②フロント制作集客用のLP・Webサイトを作るClaude Code / Codex等のAIコーディングツール
③メール一次対応問い合わせメールを検知・分類し、返信文とアポ日程を提案するGmail API、LLMによる分類・文面生成、カレンダー連携
④提案資料生成ヒアリング内容から商談用スライド・提案資料を自動生成するGamma等の資料生成AI
⑤オンライン商談Zoom / Google MeetでAI（ChatGPT Live）が声で商談を担当するGPT-Live、Realtime API、会議ボットAPI
⑥商談後処理録音・文字起こしから議事録を作成し、見積書を自動生成する文字起こし＋LLM要約
⑦フォロー対応見積送付後のやり取り・条件調整をメールで継続するLLMによるメール文面生成
⑧受注クロージングし、契約・発注につなげる人間の最終判断

すべてのステップが同じ完成度でAI任せにできるわけではない

  • ①②④はすでに実務レベルで使える段階
  • ③⑦は「仕組みとしては組める」段階。本資料ではアーキテクチャとして解説し、特定のメールアドレスへの実接続はしていない
  • ⑤⑥は今回GPT-Liveの登場でようやく現実的になった部分で、本資料で実際に動くものまで用意している
  • ⑧の最終判断は今のところ人間の仕事
2

ステップ①②④ — 商品設計・フロント・提案資料生成

①

商品設計

ChatGPTやClaudeに相手役・壁打ち相手になってもらいながら、「どんな悩みを持つ人に」「何を」「いくらで」提供するかを固める。在庫を持たない知見型の商品ほど、壁打ちだけで骨子まで作れる。

②

フロント制作

Claude CodeやCodexのようなAIコーディングツールに、商品とトーンを伝えるだけでLPやサイトのデザインから実装まで任せられる段階。問い合わせフォームから次のステップへの導線を作っておく。

④

提案資料生成

ヒアリングメモや問い合わせ内容をGamma等に渡し、商談専用の提案資料に組み替えてもらう。ゼロから作らせず「素材を渡して整形させる」のがポイント。

このあたりは既にAI活用の定番領域のため、本資料では深追いせず、次の「メールが届いてから」の部分に紙面を割きます。

3

ステップ③ — メールの一次対応

問い合わせメールが届いてからの一次対応は、次の4つの要素の組み合わせで設計できます。

  • 受信の検知：Gmail APIのプッシュ通知、またはZapier・Makeのようなノーコード連携
  • 内容の分類：メール本文をLLMに渡し、新規の問い合わせか・既存案件の返信か・クレームかを分類
  • 返信文の生成：分類結果に応じて、商品情報や過去のやり取りを踏まえた返信文を下書きさせる
  • アポの提案：Google Calendar API等で空き時間を取得し、返信文に候補日時を差し込む

この章はアーキテクチャの解説です

本資料ではこの部分を仕組みの解説にとどめ、特定のメールアドレスへ実際に接続する自動化は行っていません。メールの自動返信は一度動かすと止めにくい「常時稼働の仕組み」であり、誤送信や誤判定のリスクをその場で検証しきれないためです。実際に組む場合は、最初は「下書きまで自動生成し、送信は人間が確認してから押す」半自動の運用から始めることをおすすめします。

ステップ⑥⑦⑧ — 商談後処理から受注へ

  • 1商談中の音声・文字起こしをLLMに渡し、「議事録」（日時・参加者・議題・決定事項・次のアクション）と「見積書のドラフト」を自動生成する。見積もりはあくまでドラフトであり、金額の最終確認は人間が行う
  • 2見積送付後のフォローも、メール一次対応（第3章）と同じ設計思想で対応する。受信内容をLLMに分類・要約させ、返信文の下書きを作らせる。値引き交渉や契約条件のような繊細な判断は、最終的な送信判断を人間が行う
  • 3受注（契約・発注の確定）は、現時点では人間の最終判断に委ねる領域として扱う。ここまでAIが用意した情報（提案資料・商談の記録・見積もり）をもとに、人間が意思決定するのが今、現実的なライン

本格連携 — Zoom/Google Meetの実会議にAIを参加させる

デモはわかった、実際の商談で使いたい、という方向けに、実際に動くサーバーコードを用意しました。

静的なWebページ（第6章のai-meeting.html）だけでは、Zoom・Google Meetの実際の会議室にAIを参加させることはできません。会議室にボットとして入るには、別途サーバー側の処理が必要です。

使う技術：Recall.aiのMeeting Bot API

Recall.aiは、Zoom・Google Meet・Microsoft Teamsなどにボットとして参加し、音声・映像・文字起こしをリアルタイムで扱えるAPIを提供しているサービスです。料金は1時間あたり0.5ドル（ボットが会議に参加している時間の従量課金）で、無料トライアルもあります。

  • 1第6章の「AI商談エンジン」（ai-meeting.html）を、Recall.aiのボットが読み込む「顔」として使う
  • 2ボットのブラウザ環境内でgetUserMediaを呼ぶと会議の音声（人間の発言）を取り込める。これをそのままOpenAI Realtime APIに渡す
  • 3AIの返答音声がページ内で再生され、それがボットの出力としてそのままZoom・Google Meetの会議室に流れる
  • 4Recall.aiはリアルタイムの文字起こしをWebhookでサーバーに送ってくるので、それを受け取り蓄積し、商談終了時に議事録・見積もり生成へ渡す
配布するサーバーコード

このアーキテクチャを実際に動かすための最小構成のサーバーコード（Node.js／Express。常駐プロセスを動かせるホスティングにそのままデプロイできる形）をプレゼントキットに収録しました。実際にnode index.jsで起動確認済みです。

  • POST /api/create-bot — 会議URLを受け取り、Recall.aiにボット作成をリクエストする（Recall.aiのAPIキーはサーバー側にだけ置き、ブラウザには出さない）
  • POST /api/recall-webhook — Recall.aiからのリアルタイム文字起こしを受け取り、商談セッションごとに保存する
  • GET /api/session/:id — ボット用ページが自社ノウハウ・会社名を読み込むための参照エンドポイント
  • GET /api/realtime-token — OpenAIの通常APIキーをブラウザに渡さず、短命なclient_secretだけを発行する
  • POST /api/generate-minutes — 保存した文字起こしをOpenAIに渡し、議事録・見積もりを生成する

onepager単体では実会議への参加はできません

実際に動かすには、ご自身でRecall.aiのアカウントを作成してAPIキーを取得し、このサーバーコードをデプロイする必要があります。当方が代わりにアカウントを作ることはできませんが、コードとセットアップ手順はプレゼントキットにすべて置いてあります。

コラム — X上で報告されている音声AIの使い方

GPT-Live発表後、X上ではさまざまな使い方の報告・構想が見られました（個人の報告であり公式統計ではない点に注意して読んでください）。

  • 営業の壁打ち・ロールプレイ：価格交渉や反論処理を声でシミュレーションし、終了後に改善点をフィードバックしてもらう
  • 国際商談でのライブ翻訳：非英語圏クライアントとの通話をリアルタイム翻訳しながら進める（公式Xでも「フルデュプレックス構造により、より自然な会話・時間感覚・そしてライブ翻訳まで可能」と明言）
  • コールド架電の実験報告：GPT-Live以前の音声モードを使った実験で「2〜3%の商談化率だった」という報告もあり、より自然な会話ができるGPT-Liveでの改善が期待されている
  • 同時通訳・アンビエント通訳：会議・旅行・多言語通話での活用
  • 現場・移動中のハンズフリー業務：図面仕様の照会、運転中の口述メモなど

一方で「割り込みが多すぎて話しづらいことがある」という声も一部にあり、人間らしくなったからこそ人間らしい欠点も目立つ、という段階です。

知っておきたい注意点

  • メールの自動返信・自動送信は一度に全自動化しない。まずは「下書き自動生成、送信は人間が確認」の半自動運用から始めるのが安全
  • 商談の録音・AIによる書き起こしには相手の同意が必要な場面が多い。録音・AI利用をしていることを事前に明示する運用にする
  • 見積書の自動生成はドラフト扱い。金額の最終確認は人間が行う
  • APIキーの扱い：本資料のデモはOpenAI・Recall.aiのAPIキーをその場で入力する形式。通常のAPIキーはサーバー側で使うことが公式に推奨されている。利用上限（スペンドリミット）を設定した専用キーを使い、共有PC・公開Wi-Fiでは使わない
  • 営業電話・メールへのAI活用は法規制の対象。特定商取引法の表示義務や、個人情報保護法上の同意取得など、日本国内の規制を踏まえた運用設計が必要
  • GPT-Live自体の制限：初期リリースではビデオ・画面共有との同時利用は不可、日本語は「非ネイティブなアクセントや流暢さの差がありうる」と公式が認めている。Business/Enterprise/Eduワークスペースは初期非対応と報じられている
  • Recall.aiの利用にはコストがかかる（1時間あたり0.5ドルの従量課金）
