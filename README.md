## 1. まず結論 — 2026年7月8日、「声のAI」の当たり年が来た

先に一番大事なことを言います。**AIとの「会話」が、「もう人間と話しているみたい」という感想がX上に並ぶ水準まで来ました。**

2026年7月8日（現地時間）、OpenAIは新世代の音声モデル「**GPT-Live**」を発表し、その日のうちにChatGPTへの展開を始めました。翌7月9日には有料プラン（Go / Plus / Pro）への展開完了が[公式Xで報告](https://x.com/OpenAI/status/2075019750569378007)され、無料ユーザーへの展開も進行中です。つまりこの資料を読んでいる時点で、ほとんどの人が**すでに使える**状態にあります。

「音声モードなら前からあったでしょ？」と思った方。そのとおりで、ChatGPTにはAdvanced Voice Mode（高度な音声モード）という機能がありました。ただ、あれは正直に言えば「順番に話すトランシーバー」でした。こちらが話し終わるのを待って、AIが話し始める。途中で口を挟むと変な間ができる。会話というより「音声入力と音声読み上げの往復」だったわけです。

GPT-Liveは仕組みから違います。**聞くことと話すことを同時に行う**「フルデュプレックス（全二重）」という構造で作られていて、こちらが話している最中に「うんうん」と相づちを打ち、途中で割り込めば即座に話を止めて聞き役に回り、考える時間が必要なら黙って待ってくれます。Impress Watchの報道によれば、モデルは「1秒間に何度も『話すか、聞き続けるか、一時停止するか、割り込むか、ツールを呼び出すか』を判断」しているとのこと。これは、私たちが友人と電話で話しているときに脳がやっていることに、かなり近い動きです。

そして、これが本資料の核心ですが——**会話がここまで自然になったということは、「声でやりとりすることが本体である仕事」は、原理的にAIで代行できるようになった**ということです。営業電話、電話代行、受付、カスタマーサポート、相談窓口、通訳。この資料のテーマである「営業・電話代行」は、その巨大な変化の分かりやすい入り口にすぎません。

## 3. 何がそんなに新しいのか — 3つの柱

### 柱① フルデュプレックス — 「順番に話す」から「同時に話す」へ

従来の音声AI（Advanced Voice Modeを含む）は、大きく「カスケード型」か「ターン制」で動いていました。音声を文字に起こし、テキストで考え、答えを音声合成する。この必然として「相手が話し終わるまで待つ」構造になり、応答のたびに間が生まれ、割り込みが苦手でした。

GPT-Liveのフルデュプレックスは、**マイクの入力とスピーカーの出力が常時同時に走っている**構造です。人間の電話と同じで、聞きながら「ええ」「なるほど」と声を返し、相手が話し終わる前に反応の準備を始め、こちらの発話中に相手が割り込めば即座に譲る。報道によれば、話す・聞く・黙る・割り込む・ツールを呼ぶという判断を毎秒何度も行っています。「AIと話している感じがしない」という感想の正体はこれです。

さらにこの構造の副産物が**時間感覚**です。「ちょっと考えさせて」と言えば黙って待つ。「30秒後にもう一回言って」が通じる。ターン制のAIには原理的に難しかった「会話の間（ま）」を扱えるようになりました。

### 柱② 人格と会話の質 — 「楽しく話せる」を性能として設計

OpenAIは「良い音声モデルは話していて楽しいべき」とわざわざ公式スレッドで宣言しました。実際、人間評価では**GPT-Live-1の会話は75.7%の確率でAdvanced Voice Modeより好まれた**と報じられています。無料ユーザー向けの軽量版GPT-Live-1 miniですら69.2%。声そのものも9種類（Arbor、Breeze、Cove、Ember、Juniper、Maple、Sol、Spruce、Vale）がリマスターされ、息継ぎやイントネーションが調整されています。

「話していて楽しい」は雑談のための贅沢機能ではありません。後述する営業・相談・接客の代行では、**会話の心地よさがそのまま成約率や顧客満足になる**ため、ここが実用性の本丸です。

### 柱③ 推論委譲 — 雑談の顔をして、裏でGPT-5.5が働く

音声モデルの弱点は「賢さ」でした。速く応答するために小さいモデルを使うと、答えが浅くなる。GPT-Liveの答えは分業です。**会話のテンポはGPT-Liveが守り続け、難しい質問が来たら裏側でGPT-5.5などのフロンティアモデルに仕事を投げ、結果が返ってきたら会話に自然に織り込む。**「ちょっと調べますね——（雑談を続けながら）——お待たせしました、分かりましたよ」という、優秀なコンシェルジュの動きです。

推論の深さは Instant / Medium / High の3段階が用意されていると報じられており、雑談なら即答、契約書の読み解きのような重い相談なら深く考える、という使い分けが利きます。天気・株価・スポーツなどは音声と並行して**視覚カード**が画面に出るため、「声だけでは伝わりにくい情報」もカバーされます。

## 4. 数字で見るGPT-Live — 「音声なのに賢い」の意味

公式ブログの数値として各メディアが報じているベンチマークを整理します。比較対象はいずれも従来のAdvanced Voice Mode（AVM）です。

| 指標 | Advanced Voice Mode | GPT-Live-1（High） | 何を意味するか |
|---|---|---|---|
| 会話選好（人間評価） | — | **75.7%が GPT-Live を選好**（miniは69.2%） | 話し相手としての快適さ |
| GPQA（大学院レベル科学推論） | 45.3% | **84.2%** | 会話しながらの「頭の良さ」が約1.9倍 |
| BrowseComp（Web検索タスク） | 0.7% | **75.2%** | 「調べながら話す」が実用になった |
| τ³-Voice Telecom(顧客サポート模擬) | — | 全GPT-Live版がAVMを上回ると報告 | 電話業務の実務適性 |

特筆すべきはBrowseCompです。0.7%→75.2%という数字は「できなかったことが、できるようになった」変化であって、改善という言葉では足りません。従来の音声AIは実質「検索しながら会話する」ことができなかったのに対し、GPT-Liveは**電話しながら調べ物をして戻ってくる**。これは営業や問い合わせ対応で「確認して折り返します」が消えることを意味します。

料金は単体課金ではなく、ChatGPTの既存プランに含まれる形です（報道ベースの整理）。

| プラン | 月額 | 音声の標準モデル |
|---|---|---|
| Free | $0 | GPT-Live-1 mini |
| Go | $8 | GPT-Live-1 |
| Plus | $20 | GPT-Live-1 |
| Pro | $200 | GPT-Live-1（利用上限が大きい） |

無料でもminiが使える、つまり**新世代の音声会話の入場料は実質ゼロ**になりました。ここも普及速度を考えるうえで大きな要素です。

## 5. 「声の仕事」がAIに置き換わる — 営業・電話代行はその一例

ここからが本題です。GPT-Liveの登場が示したのは「ChatGPTの音声モードが良くなった」ではなく、**「声のやりとりそのものが業務の本体である仕事は、AIが実用水準で担えるようになった」**という構造変化です。

考えてみてください。営業電話・電話代行・受付・サポート窓口の仕事を分解すると、中身はこうです。

1. **聞く** — 相手の話を遮らず、相づちを打ちながら要点を掴む
2. **即応する** — 不自然な間を作らず、テンポよく返す
3. **調べる** — 分からないことを確認して、正確に答える
4. **記録する** — 会話の内容を整理して次のアクションにつなぐ

この4つのうち、①と②はフルデュプレックスが、③は推論委譲とWeb検索が、④はテキスト化と要約がそれぞれ直撃しています。**「電話の仕事」の構成要素が、今回の発表で全部そろった**わけです。

営業電話で言えば、X上ではGPT-Live以前の音声モードを使ったコールド架電の実験で「2〜3%の商談化率だった」という報告が見られました（個人の報告であり公式統計ではありません）。ターン制のぎこちない音声ですらその数字が出ていたなら、相づちと割り込みを備えた自然な会話でどうなるか——期待が集まるのは自然な流れです。電話代行についても、τ³-Voice Telecom（通信業の顧客サポートを模したベンチマーク)で全GPT-LiveモデルがAVMを上回ったという報道は、「一次受けをAIに任せる」判断を後押しする材料になります。

ただし、誠実に付け加えるべきことが2つあります。第一に、**ChatGPTアプリのGPT-Liveを直接「自社の電話番」にはまだできません。** APIは「近日提供」の段階で、電話システムに組み込むには既存のRealtime API（gpt-realtime系）を使う必要があります（第9章）。第二に、日本では**特定商取引法や個人情報の扱いなど、営業電話に固有のルール**があり、「AIだから何をしてもいい」わけではありません。置き換えの波が来ていることと、運用に設計が要ることは、両方本当です。

## 6. 事例集 — X上で報告されている使い方

発表から48時間で、X上にはすでに多様な使い方の報告・構想が並んでいます。hermes（X検索）で拾い集めた中から、動画の構成に使える事例を整理します。いずれも個人ユーザーの報告・構想ベースであることを踏まえて読んでください。

**仕事を「代行」する方向の事例:**

- **営業の壁打ち・ロールプレイ** — 価格交渉や反論処理を声でシミュレーション。AIが顧客役として反問し、終了後に改善点を口頭でフィードバック。従来のテキスト壁打ちと違い「間」や「言い淀み」まで練習できる
- **商談・通話のリアルタイム支援** — 人間が話している通話の横でAIがメモを取り、必要な情報を出す「サイドパイロット」構想。コンタクトセンターの一次応答補助として企業の関心が報告されている
- **国際営業・海外フリーランス** — 非英語圏クライアントとの通話をライブ翻訳しながら進め、言い回しの推敲まで頼む。「現地語ができないから諦めていた市場」の開拓
- **相談・カウンセリング的な使い方** — 悩みを声で吐き出し、整理してもらう。テキストより感情が乗る分、「聞いてもらえた」感覚が強いという報告

**「声がインターフェースになる」方向の事例:**

- **同時通訳・アンビエント通訳** — 会議、旅行、多言語通話。「英語⇔日本語を双方向で通訳して」と頼むだけで通訳者モードになる
- **割り込めるニュース番組** — 朝のニュースを読み上げてもらい、気になったところで「待って、それ詳しく」と割り込む。裏でWeb検索が走って深掘りが返ってくる
- **現場・運転中のハンズフリー業務** — 建設現場を歩きながら図面仕様を照会、運転中に報告書を口述、移動時間がそのまま作業時間になる
- **語学・面接練習** — 相づちも割り込みもある「本物の会話」での練習。発話の途中で言い直しても文脈が壊れない

一方で、**「割り込みが多すぎて、かえって話しづらいことがある」という否定的な感想も一部にあります。**編み物をするおばあちゃんとの公式デモ動画に対しても、自然さを称賛する声と「人の話を遮りすぎ」という指摘の両方がありました。人間らしくなったからこそ、人間らしい欠点（話の遮り方の癖）が目立つ、という段階です。カスタム指示での調整や今後のアップデートで変わっていく部分でしょう。

## 7. キーボードを叩かない未来 — 会話がエージェントの操縦桿になる

もう一段引いて考えます。私たちがAIに指示を出すとき、これまでの入力装置は**キーボード**でした。プロンプトを書き、結果を読み、また書く。この「書く→読む」の往復が、AI活用の速度を決めていました。

GPT-Liveが示したのは、この操縦桿が**声**に替わる未来です。ポイントは、単なる音声入力（書く代わりに喋る）ではないことです。フルデュプレックスの会話では、指示→確認→修正→実行のループが**会話のテンポ**で回ります。「あ、違う、そうじゃなくて」と割り込んだ瞬間に軌道修正が始まる。推論委譲の仕組みは、まさに「会話しながら裏でエージェントが動く」構造そのものです。実際の発表でも、GPT-Liveは検索や複雑な作業をフロンティアモデルに投げて、終わったら会話に持ち帰ると説明されています。**話し相手が、そのまま仕事の発注窓口になっている**わけです。

すでに周辺の動きも揃い始めています。開発の現場では、superwhisperやWispr Flowのような音声入力ツールでコーディングエージェントに口頭指示を出すスタイルが広がっていますし、OpenAIのRealtime APIは音声エージェントを電話網(SIP)につなぐ拡張を進めてきました。「AIに何かを頼む＝キーボードに向かう」という等式は、この1〜2年で崩れていくと考えるのが自然です。

動画でお伝えしたい未来像はこうです。**朝、歩きながら「昨日の続きのリサーチ、まとめておいて。あと午後の商談の想定問答を作って、終わったら電話して」と話す。昼、AIから電話が来て、口頭で修正を伝える。夕方には成果物ができている。**——GPT-Liveと既存のエージェント技術を素直につなぐと、これは絵空事ではなく、設計図が描ける段階の話です。

## 8. 使い方（初心者向け） — 今日から試す手順

難しい設定は何もありません。「アプリを最新にして、マイクのボタンを押す」。これだけです。

1. **ChatGPTアプリを最新版に更新する**（iOS / Android。Webはそのままでも可）
2. **入力欄の右にある音声（Voice）ボタンをタップ**する
3. 話しかける。**途中で割り込んでいいし、「ちょっと待って」と言えば待ってくれます**
4. 終わるときは「会話を終了して」と言うか、終了ボタンをタップ

コツは、テキスト時代の「プロンプト」を忘れることです。命令文を組み立てるのではなく、**人に頼むように話す**方がうまくいきます。悩み別の「最初のひとこと」を用意しました。

| こんなとき | こう話しかける |
|---|---|
| 何ができるか知りたい | 「GPT-Liveになって何が変わったの？デモっぽく見せて」 |
| 営業の練習がしたい | 「あなたは値下げ要求の厳しいお客さん役。私が営業。途中で遠慮なく割り込んで」 |
| 通訳してほしい | 「今から英語と日本語の同時通訳者になって。私が日本語で話したら英語に、相手の英語は日本語にして」 |
| 相談に乗ってほしい | 「今日あったことを整理したいから、聞き役になって。ときどき質問して」 |
| ニュースが聞きたい | 「今朝のAI関連ニュースを番組みたいに読み上げて。気になったら割り込むね」 |

この「最初のひとこと」を100本に増やしたものを、プレゼントの**ChatGPT Live活用術100選**として配布します（第11章）。

## 9. もう一歩進んだ使い方 — 開発者向けの現在地

自社サービスや電話システムに組み込みたい人向けの整理です。

- **GPT-LiveのAPIは未提供**（公式に「Coming soon to the API」）。ChatGPTアプリ内での体験が先行しています
- いま開発者が使えるのは **Realtime API**（gpt-realtime系モデル）。WebRTC/WebSocketで音声をストリーミングし、音声エージェント・電話統合(SIP)・リアルタイム翻訳などが構築できます。GPT-Liveで示された方向性を、API側で先取りして組む形です
- 本資料のプレゼント第3弾**「リアルタイム翻訳サイト」**は、まさにこのRealtime APIをブラウザから直接叩く構成で作りました。OpenAIのAPIキーを入れるだけで、マイクの音声を選んだ言語に通訳して読み上げます。中間サーバーを置かず、ブラウザからOpenAIへ直接接続する構成で、コードも公開しています。なお通常のAPIキーは本来サーバー側で使うことが公式に推奨されているため、利用上限を設定した専用キーでの個人利用を前提に、注意点を配布ガイドにまとめています（[translate.html](https://fuuuuuuma.github.io/gpt-live-guide-ja/translate.html)）
- ChatGPT側のGPT-Liveがビジネス/エンタープライズ/教育ワークスペースで初期非対応と報じられている点も、業務導入の際は確認してください

## 10. 知っておきたい注意点

誇張なしで使うために、現時点の制限・不確定要素をまとめます。

- **初期リリースではビデオ・画面共有との同時利用は不可**と報じられています（今後対応予定）
- **日本語の流暢さ**について、公式は「一部の言語では非ネイティブなアクセントや流暢さの差がありうる」と認めています。日本語での実務投入前に自分の用途で試すのが安全です
- ロールアウト初期には、一部有料ユーザーに**1日あたり約1時間程度の利用制限**があるという報道があります（流動的な情報です）
- 音声会話のデータは**既定でAI学習にオプトアウト**され、30日保持・手動削除が可能と報じられています。とはいえ、商談内容など機密性の高い会話を通す際は所属組織のルールを確認してください
- 営業・勧誘の電話にAIを使う場合、**特定商取引法など日本の法規制と、通話録音・本人明示の運用設計**が必要です。「技術的にできる」と「そのまま業務に使ってよい」は別物です
- 会話の自然さには個人差のある評価があり、「割り込みが多い」と感じる報告も存在します
